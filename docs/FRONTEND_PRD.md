# [PRD] 카드 추천 & 비교 서비스 프론트엔드 명세 (Desktop Web)

## 1. 서비스 개요 및 디자인 원칙
- 목표: 유저 소비 패턴 분석을 통한 최적의 카드 추천 및 기존 카드와의 혜택 비교
- 플랫폼: 데스크탑 웹 전용 (Desktop-First)
- 디자인 컨셉: 캐릭터 배제, 신뢰감 있는 금융 대시보드 스타일
- 가독성: 핵심 수치는 28px~36px의 큰 폰트 사용, 일반 본문은 18px 이상
- 공간 활용: 2분할(Split) 레이아웃을 통해 스크롤 최소화 및 한눈에 들어오는 정보 배치
- 정보 제어: 상세 정보는 토글(Toggle)을 활용하여 초기 복잡도 감소

## 2. 전역 상태 및 데이터 구조 (Global State)
Codex는 아래 데이터를 기반으로 각 단계를 관리해야 합니다.

- comparisonMode: "COMPARE" (기존 카드와 비교) | "NEW" (새 카드 추천만)
- selectedCurrentCard: 기존 카드 정보 (ID, 이름, 카드사, 이미지, 타입)
- selectedCategories: 유저가 선택한 혜택 카테고리 (Array, 2~5개)
- spendingData: 카테고리별 월 소비 금액 (Map/Object)

## 3. 화면별 상세 명세 및 분기 로직

### Step 1. 서비스 모드 선택 (Entry)
- 경로: /
- 선택 옵션:
- [사용 중인 카드와 비교해서 추천받기] -> 클릭 시 comparisonMode = COMPARE 설정 후 Step 2로 이동
- [새로운 카드 중에서 추천받기] -> 클릭 시 comparisonMode = NEW 설정 후 Step 3으로 즉시 이동 (Step 2 스킵)

### Step 2. 기존 카드 선택 (Card Selector - 분기형)
- 경로: /select-card
- 핵심 컴포넌트: RichCardSearchSelect (검색 가능한 드롭다운)
- 데이터 노출 항목: 카드 이미지 썸네일, 카드 상품명, 카드사 이름, 카드 타입 뱃지(신용/체크)
- 로직: 우리가 보유한 카드 데이터 내에서만 선택 가능. 선택 완료 시 '다음' 버튼 활성화

### Step 3. 혜택 카테고리 선택 (Categories)
- 경로: /categories
- 레이아웃: 5열 그리드 타일 시스템
- 요건: 아이콘 + 카테고리명. 2~5개 선택 시에만 '다음' 버튼 활성화

### Step 4. 소비 금액 입력 (Spending Input)
- 경로: /input-spending
- 레이아웃: 2-Column Split (가로 분할)
- 좌측 (고정/Sticky): Real-time Chart Section. 도넛 차트와 총 소비액 합계를 노출. 우측 입력값 변경 시 실시간 애니메이션 반영
- 우측 (스크롤): Input Controller Section. 각 카테고리별 [슬라이더 + 숫자 Input Box + 상세 설명 토글] 배치. 텍스트와 컨트롤러 크기를 큼직하게 구성

## 3-1. 화면별 UX 디테일 가이드 (Gemini)

### Step 1. 서비스 모드 선택 (Entry)
- 헤더: "선호하시는 서비스를 선택해주세요" (32px 이상, Bold)
- 선택 카드 (2개): "기존 카드 비교"와 "새 카드 추천"을 큰 카드 형태의 버튼으로 배치
- 인터랙션: 마우스 호버 시 카드의 테두리 색상 변화 또는 살짝 떠오르는 효과로 클릭 가능성 강조
- 분기 로직: 선택 값에 따라 isComparison 상태 업데이트 및 다음 경로 결정

### Step 2. 기존 카드 선택 (Card Selector)
- 검색창 결합형 드롭다운: 텍스트 입력 시 실시간 필터링되는 Combobox 형태
- 리스트 아이템: 카드 이미지 썸네일 + 카드명 + 카드사명 + 신용/체크 타입 뱃지
- 데이터 범위: 서비스 보유 카드 DB 내에서만 선택 가능하도록 제한
- 시각적 피드백: 선택된 카드는 드롭다운 상단에 이미지와 함께 확정된 형태로 표시

### Step 3. 혜택 카테고리 선택 (Category Grid)
- 5열 그리드: 아이콘과 텍스트가 결합된 타일(Tile) 형태의 버튼 배치
- 하단 가이드: "2~5개를 선택해 주세요" 안내 문구를 버튼 바로 위에 배치
- 다중 선택 로직: 클릭 시 상태 토글, 선택된 타일은 포인트 컬러(Blue)로 강조
- 유효성 검사: 선택 개수가 2~5개 범위를 벗어나면 '다음' 버튼 비활성화

### Step 4. 소비 금액 입력 (Spending Dashboard)
- 좌측 (Fixed/Sticky): 실시간 도넛 차트와 총 소비 금액 수치 노출
- 우측 (Scroll): 카테고리별 입력 섹션, 슬라이더와 숫자 입력창 병렬 배치
- 실시간 연동: 우측 슬라이더 조절 시 좌측 차트 비율과 총액이 즉각적으로 변하는 애니메이션 적용
- 정보 제어 (Toggle): 부가 설명은 기본 숨김, 클릭 시 펼쳐지는 아코디언/토글 사용

### Step 5. 최종 결과 및 비교 화면 (Output)
- 비교 모드: 좌측 기존 카드 고정, 우측 추천 카드 캐러셀, 혜택 차액(+N만원) 강조
- 추천 모드: 3개 카드를 가로 그리드로 배치하여 한눈에 비교 가능하게 구성
- 사이드 패널: 카드 하단 버튼 클릭 시 우측 45% 영역에서 상세 질의 패널 슬라이드 인
- AI 큐레이션: 유저 소비 습관을 요약하는 텍스트 카드 배치

## 4. 결과 출력 화면 명세 (Output)

### Case A: 새로운 카드 추천 (New Mode)
- 레이아웃: 3열 그리드 (1, 2, 3순위 카드 나열)
- 카드 구성: 연간 예상 혜택 금액 강조, 카테고리별 혜택 상세 리스트, '더 물어보기' 버튼

### Case B: 기존 카드와 비교 (Compare Mode)
- 레이아웃: 2-Part Layout
- 좌측 (고정): 기존 카드 정보 요약 및 "바꾸면 연간 N만원 더 이득" 메시지 강조
- 우측 (캐러셀): 추천 카드 1, 2, 3순위를 가로 캐러셀로 배치. 1순위 카드를 시각적으로 가장 크게 강조
- 특이사항: 추천 카드 리스트에는 기존 카드 대비 추가 혜택액(+N만원)을 하이라이트

## 5. 인터랙션 요구사항: 상세 질의 패널 (Side Sheet)
- 트리거: 모든 결과 화면의 카드 하단 '더 물어보기' 버튼 클릭
- UI: 우측에서 슬라이드하며 나타나는 45% 너비의 사이드 패널
- 기능: 선택된 카드 정보와 유저 입력값을 기반으로 한 질문 칩(Chip) 제공
- 채팅 인터페이스를 통한 상세 답변 제공 (패널 내부 개별 스크롤)
- 닫기(X) 버튼 제공 및 패널 오픈 시 메인 화면 배경 Dim 처리

## 6. 기술적 제약 사항 (Codex 지시용)
- TDD 개발: 모든 컴포넌트와 비즈니스 로직(혜택 계산 등)에 대해 테스트 코드를 먼저 작성할 것
- 컴포넌트화: 모든 화면 요소를 원자 단위로 쪼개어 재사용 가능하게 설계할 것
- 데이터 바인딩: 입력 단계의 모든 상태가 결과 페이지의 계산 로직 및 사이드 패널의 컨텍스트로 정확히 전달되어야 함
