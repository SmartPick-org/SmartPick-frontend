# Worklog: Add card list API

## 왜 이렇게 했나요
- 카드 선택 화면에서 실제 데이터 기반 목록이 필요합니다.
- 현재 추천 API는 있지만 카드 목록을 내려주는 API가 없어 프론트가 mock에 의존하고 있었습니다.

## 무엇을 수정했나요
- `GET /cards` 엔드포인트를 추가했습니다.
- `json_v3` 데이터셋을 읽어 카드 기본 정보를 리스트로 반환합니다.
- digest 데이터에서 첫 줄을 추출해 간단 설명(`digest_summary`)으로 제공합니다.

## 확인 방법
- 요청: `GET /cards`
- 응답: 카드 기본 정보 리스트와 요약 설명 포함

## 참조 주석
- 엔드포인트: 서버에서 요청을 받을 수 있는 URL 경로
- digest: 카드 혜택 요약 문서(마크다운)
