# Worklog: Input spending layout polish

## 왜 이렇게 했나요
- 소비 입력 단계는 정보량이 많아서 좌/우 분할 구조가 확실해야 합니다.
- 도넛 차트는 추후 연결 예정이라 기본형 자리만 먼저 확보했습니다.
- 서버 Enum을 기준으로 저장하므로, 화면에서는 한글 라벨로 보여주도록 매핑이 필요합니다.

## 무엇을 수정했나요
- 좌측 요약 영역에 차트 placeholder와 총액 카드를 강조했습니다.
- 우측 입력 영역의 카드 레이아웃과 간격을 정리했습니다.
- 카테고리 라벨을 영문 Enum → 한글로 변환해 표시했습니다.

## 확인 방법
- 실행 경로: `/input-spending`
- 기대 동작: 한글 라벨로 표시되고, 슬라이더/입력이 정상 동기화됩니다.

## 참조 주석
- placeholder: 실제 기능이 들어올 자리를 미리 표시하는 임시 UI
- 매핑: 서로 다른 표기(영문/한글)를 연결하는 표
