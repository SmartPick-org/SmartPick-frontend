# Worklog: Connect select card to API

## 왜 이렇게 했나요
- 카드 목록 API(`GET /cards`)가 생겨서 실제 데이터로 카드 선택 화면을 채울 수 있습니다.
- API 실패 시에도 화면이 비지 않도록 임시 데이터(폴백)를 유지했습니다.

## 무엇을 수정했나요
- 카드 목록을 `GET /cards`에서 받아오도록 연결했습니다.
- 카드 정보 타입을 정의하고 검색 로직을 카드명/카드사 기준으로 개선했습니다.
- digest 요약이 있으면 카드 설명으로 표시하도록 추가했습니다.

## 확인 방법
- 실행 경로: `/select-card`
- 기대 동작: 서버가 켜져 있으면 실제 카드 목록이 보이고, 꺼져 있으면 기본 카드가 보입니다.

## 참조 주석
- 폴백: 네트워크 실패 시 대신 쓰는 기본 데이터
- API: 서버에 데이터를 요청하는 통신 경로
